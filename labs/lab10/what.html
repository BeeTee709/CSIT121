Overview

IndexedDB API 2.0 released in 2018

2.0 is the current version of the webstandard that is recommended for use by the W3C. The original version was released in 2010.

IndexedDB API is "low level API for client-side storage of a significant amount of structured data, including files/blobs" (Mozilla Contributors, 2025). This means IndexedDB acts as a storage unit built into your browser that also allows provides a way for web applications to store data persistently, even when the user is offline, and allows for complex querying and indexing of that data.

Improvements have been made by way of error handling and overall ease of use. Including adding promises, which makes it easier to work with asynchronous operations. Something that is heavily relied on in modern web development. Especially in API development,

IndexedDB is other option than localStorage or Cookies for client-side storage.
